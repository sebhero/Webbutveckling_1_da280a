  1. Förklara kort vad ett element, attribut och värde är för något

    Element är ett sätt att ordna innehållet i ett html dokument. tex <strong> </strong> är ett element. alla element har en start tag och en matchande sluttag. Ett element har attributer tex. ett img element består av attributerna src och alt.
    Dessa attributer är länken till bild filen och alternativ text ifall man kunde inte läsa in bilden i webläsaren. Element är vad webläsaren läser av och omvandlar till en websida och attribut är inställningarna man har satt på de element som var bilden finns, storleken på bilden m.m. värde är tex. hur bred bilden ska vara. som man kan sätta medhjälp av width="värde"


  2. Förklara vad HTTP är för något och vad ett kommunikationsprotokoll innebär

    HTTP står för Hyper Text Markup Language. som är språket man använder för att göra websidor. för att kunna visa en websida på internet måste man använda sig av HTTP som säger vilken typ av kommunkitions protokoll man ska använda. Andra kommunikations protokoll är tex. TCP/IP (nätverks kommunikation) eller STMP (epost).


  3. Vilka är W3C och vilken är deras huvudsakliga uppgift?

    W3C är ett industrikonsortium bestående av olika företag, forskare och regeringar.
    Enligt wikipedia är W3C uppgift: "W3C arbetar med att utveckla tekniska protokoll, standarder och programvara för webben med målet att genom ett öppet samarbete leda Internet till dess fulla potential." (wiki)
    Deras uppgift är att internet protokoll och sätta standarden. som sedan webläsarna implementerar.


  4. Vad står WWW för och vad är detta för något?

    www står för World Wide Web.
    detta gör det möjligt att läsa in websidor från internet.

  8. Vad står HTML för och vilket typ av språk är detta?

    HTTP står för Hyper Text Markup Language. det är ett markup språk. andra exempel på markup språk är markdown. som du kan skriva websidor med eller xml som nya MS Office bygger sina dokument på.
    Markup språk skriver du i text hur texten ska formatera sig. därefter använder du en läsare (för websidor webläsare) som då tolkar texten du skrivet och omvandlar den till text och layout. därför när man glömmer tex. ">" i din html text så ser du hela html elementet med dess attributer. Detta är pga att webläsaren vet inte då vilket element du vill visa.


  9. Varför använder vi oss av indentering (indrag) när vi skriver HTML-kod?

    för att det gör det enklare för oss som utvecklare att läsa vad vi försöker göra. Det gör också enklare att hitta missar. Det ger oss struktur i våran kod. Så att det blir enkelt att följa flödet.


  10. Blir det någon skillnad på utseendet av sidan när du indenterar din HTML-kod?

    nej. indenteringen påverkar inte. För att webläsaren läser inte av indenteringen. Finns andra språk som använder sig av indenteringen för att förklara att detta är en if-sats/objekt m.m. tex Python.
    enligt kurs materialet "Med indentering menar vi att vi strukturerar upp vår källkod (HTML) på ett sätt som underlättar för oss människor att läsa koden." ( ref)


  11. Varför tror du att vi lär oss behärska HTML-kod från grunden till skillnad från att generera HTML-kod med hjälp av s.k. WYSIWYG-programvaror och HTML-generatorer?

    För att ifall man använder sig av något som generar kod. Så förstår man inte hur den fungerar. Samt så generatorer kanske inte heller generar rätt. utan ger massa extra element som inte behövs. Det blir en sallig röra.   12. Vilka två delar består ett HTML-dokument av? Vilken sorts information finns i respektive del?
    Head och Body.
    1. Head: Innehåller metadata om din websida. tex. Titeln som syns högst upp i webläsaren.
    2. Body: det är din själva websida. det som du ser. här i går den mesta av din html kod. Det är innehållet.

  13. Varför har vi s.k. HTML-kommentarer i vår kod?

    För att förtydliga vad element/bitar i våran websida gör och betyder. Det ger mer förståelse vad bitarna i websidan gör/ska göra.

    Jag har förståt det från min java kurs. Att man använder sig utav kommentarer för att förtydliga vad funktionen gör så att det blir enklare när man i framtiden tittar tillbaks på det man har skrivit och ifall någon annan ska uppdatera det du har skrivit.
